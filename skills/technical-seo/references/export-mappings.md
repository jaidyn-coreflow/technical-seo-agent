# Export Mappings: SEO Queries to Screaming Frog MCP Parameters

This reference maps common SEO query patterns to the correct `export_crawl` parameters for the Screaming Frog MCP server. When a user asks a question about their site, match their intent against the **Query Pattern** column and use the corresponding export parameters.

There are three parameter types:

- **`export_tabs`** — Named tabs from the Screaming Frog UI (format: `Category:Filter`). Multiple tabs are comma-separated.
- **`bulk_export`** — Bulk export operations that produce separate data sets.
- **`save_report`** — Summary reports generated by Screaming Frog.

Use the most specific match. If the query is broad or ambiguous, prefer the "All" filter for the relevant category. For full site audits, see the Full Audit section at the bottom.

---

## Crawlability

| Query Pattern | `export_tabs` | `bulk_export` | `save_report` |
|---|---|---|---|
| 404s, broken pages, client errors | `Response Codes:Client Error (4xx)` | | |
| Server errors, 5xx, down pages | `Response Codes:Server Error (5xx)` | | |
| Redirects, redirect chains, 301s/302s | `Response Codes:Redirection (3xx)` | `All Redirect Chains` | |
| Orphan pages, unlinked pages | | `Orphan Pages` | |
| Crawl overview, crawl summary | | | `Crawl Overview` |

## Link Structure

| Query Pattern | `export_tabs` | `bulk_export` | `save_report` |
|---|---|---|---|
| Internal links, inlinks, linking pages | | `All Inlinks` | |
| Outlinks, external links, outbound links | | `All Outlinks` | |

## Performance

| Query Pattern | `export_tabs` | `bulk_export` | `save_report` |
|---|---|---|---|
| Response time, slow pages, TTFB | `Internal:All` | | |
| Page size, large pages, bloated pages | `Internal:All` | | |

## Structured Data

| Query Pattern | `export_tabs` | `bulk_export` | `save_report` |
|---|---|---|---|
| Schema, structured data, rich results | `Structured Data:All` | | |

> **Note:** The `Structured Data` tab is only available when structured data extraction is enabled in the Screaming Frog configuration. If the export returns empty or errors, advise the user to check that **Configuration > Spider > Extraction > Structured Data** is turned on in Screaming Frog.

## Full Audit

When the user requests a comprehensive site audit, full crawl review, or general SEO health check, use all of the following parameters together:

**`export_tabs`:**
- `Internal:All`
- `Response Codes:All`
- `Page Titles:All`
- `Meta Description:All`
- `H1:All`
- `Canonicals:All`
- `Directives:All`
- `Images:All`

**`bulk_export`:**
- `All Inlinks`
- `All Redirect Chains`

**`save_report`:**
- `Crawl Overview`

---

## Notes

- **Tab name verification:** The exact tab and filter names (e.g., `Response Codes:Client Error (4xx)`) must match what the Screaming Frog MCP server expects. If an export call fails with an unrecognized tab name, query the MCP server's available exports to confirm the current naming. Tab names can vary between Screaming Frog versions.
- **Structured data availability:** The `Structured Data:All` export depends on Screaming Frog being configured to extract structured data during the crawl. This is not enabled by default. If results are missing, the crawl may need to be re-run with extraction enabled.
